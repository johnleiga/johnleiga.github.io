Production-Style ML Research Infrastructure

A modular, independently built system for large-scale equity and options data processing, feature engineering, model training, backtesting, and live-ready inference.

I built this project to demonstrate end-to-end system design rather than isolated modeling work. Instead of relying on notebooks or one-off scripts, I structured the repository like production research infrastructure: modular pipeline stages, clear data contracts, configuration-driven experiments, reproducible run artifacts, and realistic backtesting assumptions.

The system ingests stock and options data at scale, builds features across price, volatility, volume, Greeks, and term structure, generates labels for direction, magnitude, and tail risk, trains LightGBM models, evaluates strategies through structured backtests, and supports live-ready inference logic.

While the domain is quantitative research, the engineering principles are the same ones I apply in hardware: separation of concerns, scalable architecture, validation between stages, and a bias toward systems that can be extended without becoming fragile.

Project Highlights

  • Modular CLI-based pipeline
  • Configuration-driven experimentation using TOML
  • Large-dataset handling with Parquet and PyArrow
  • Multi-objective modeling using LightGBM
  • Auditable backtesting with realistic execution constraints
  • Live inference pipeline for real-time signal generation

Architecture Focus

The pipeline is separated into data ingestion, feature engineering, labeling, model training, backtesting, live inference, and explainability. Each stage is independently executable and externally configured, which makes experiments reproducible and debugging much more targeted.

Why it matters

I included this project because it shows how I think about complex systems beyond traditional mechanical engineering work. It demonstrates end-to-end ownership, architectural judgment, scalability, and the ability to build technical infrastructure from scratch around real constraints.
